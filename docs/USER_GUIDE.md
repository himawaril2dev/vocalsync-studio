## 1. 歡迎使用

VocalSync Studio 是一間**桌面練唱室**，讓想練習唱歌的你可以：

- 邊聽伴奏邊錄音，看見自己的演唱音高曲線
- 與目標旋律（原唱分離出的 vocals）並排比對
- 透過 A-B 循環反覆打磨特定段落
- 變速不變調，慢速練習高難度樂句

程式為**完全離線運作**（除下載伴奏外），錄音檔僅儲存於你的電腦，不會上傳到任何雲端。

> 📢 **透明度聲明**
> 作者沒有程式開發背景，本專案由 AI（Claude / Codex）協作完成。所有功能皆經實測與跨模型 code review，請依你的使用情境評估風險後採用。

## 2. 快速開始 5 分鐘

### Step 1｜啟動程式

解壓縮 `VocalSync.Studio.Portable.x.y.z.zip`，雙擊資料夾內的 **`vocalsync-studio.exe`**。

> ⚠️ 資料夾結構請勿變動。`DirectML.dll`、`models/` 是隨包執行期依賴，**不可單獨搬移**。自動安裝的 yt-dlp / FFmpeg 會優先放在免安裝包根目錄，整個資料夾搬到哪裡都能跑。

### Step 2｜準備一首伴奏

**選項 A：從 YouTube 下載**

1. 切到左側導覽列的「**準備**」分頁
2. 展開最上方的「**YouTube 下載**」區塊
3. 貼上 YouTube 網址（例如：`https://www.youtube.com/watch?v=xxxxxxxxxxx`）
4. 格式選 `mp3`，點「**下載**」即可

首次使用若提示缺少 yt-dlp 或 FFmpeg，點「**自動安裝**」按鈕讓程式幫你裝好。

**選項 B：使用本機音檔**

展開「**練唱伴奏**」區塊 → 點「**匯入伴奏**」→ 選擇你的 MP3 / WAV / M4A 等檔案。

### Step 3｜設定輸入輸出裝置

展開「**裝置選擇**」區塊：

- **輸入裝置**：你的麥克風
- **輸出裝置**：你的耳機（**強烈建議戴耳機錄音**，不然麥克風會收到伴奏，錄出來混在一起）

### Step 4｜校準延遲（首次使用推薦做一次）

展開「**延遲校準**」區塊 → 點「**開始校準**」→ 戴好耳機後，**跟著節拍器的木魚聲拍手**（或拍桌、用嘴發「嗒」聲都可以）。

程式會播放 8 拍（前 2 拍暖身 + 後 6 拍量測），偵測你拍手的時間點與節拍器的延遲差，自動算出最佳補償值並記住，之後不用再做（除非你換裝置）。

### Step 5｜開始錄音

1. 切到「**錄音**」分頁
2. 按空白鍵或**紅色錄音鍵**開始
3. 跟著伴奏唱
4. 按空白鍵暫停，或按 `Esc` 停止
5. 按「**匯出**」將錄音儲存為 WAV 檔

下半區的音高曲線會即時顯示你唱的音高，上方可以同步看歌詞或影片。

## 3. 主介面導覽

左側導覽列有 4 個分頁：

### 🎯 準備（Setup）

第一次開啟時停在這裡。所有**「錄音前要做好的事」**都收在這頁，用可收合的區塊組織：

- YouTube 下載
- 練唱伴奏
- 歌詞 / 字幕
- 目標旋律
- 裝置選擇
- 延遲校準

收合狀態會在切分頁後保留，不用每次重點。

### 🎤 錄音（Recording）

主要工作區。上半分為左右兩欄（可拖動分隔線調整比例）：

- **左：影片 / 音檔資訊**（若匯入影片會顯示畫面）
- **右：歌詞面板 / 歌詞同步編輯器**（可切換）

下半是**全寬的音高時間軸**，即時顯示演唱與目標旋律的對比。

最下方兩列控制列：

- **播放列**：試聽 / 暫停 / 停止 / 錄音 / 進度條 / A-B 循環
- **控制列**：伴奏音量 / 麥克風音量 / 變速 / 變調 / 匯出 / 回放

### 📊 音高（Pitch）

專注於音高分析的分頁。上方顯示當前 melody 狀態與對齊結果，中間是全螢幕的音高時間軸，下方有迷你播放列可邊聽邊看。

若你只想不錄音、單純回看上次錄音曲線與目標旋律的對比，可在這分頁操作。

### ℹ️ 關於（About）

版本資訊、透明度聲明、授權、贊助連結。

### 語言切換

左側導覽列最下方有語言切換器，支援 **繁體中文 / English / 日本語**，即時切換不需重啟。

## 4. 功能詳解

### 4.1 YouTube 伴奏下載

**位置**：準備分頁 → YouTube 下載區塊

程式**不內建** yt-dlp 與 FFmpeg，第一次使用時會偵測並顯示狀態：

| 狀態 | 意義 | 處理 |
|---|---|---|
| 🟢 yt-dlp v2026.x.x | 已安裝可用 | 無需處理 |
| 🔴 yt-dlp 未安裝 | 找不到 yt-dlp | 點「**自動安裝**」按鈕 |
| 🟢 FFmpeg ✓ | 系統有 FFmpeg | 無需處理 |
| 🔴 FFmpeg 未安裝 | 系統沒裝 | 點「**自動安裝**」按鈕 |

**工具信任與自動安裝會做什麼**：

- **使用本機工具**：可偵測你電腦上的 yt-dlp / FFmpeg；按下信任後會記錄路徑與 SHA-256，之後只執行同一個已驗證檔案
- **yt-dlp 自動安裝**：從 GitHub Releases 下載 `yt-dlp.exe`，驗證 SHA-256 後優先放在免安裝包根目錄
- **FFmpeg 自動安裝**：從固定的 essentials build 下載壓縮檔，驗證 SHA-256 後解壓 `ffmpeg.exe` / `ffprobe.exe`，優先放在免安裝包根目錄
- **備援位置**：免安裝包根目錄無法寫入時，工具會放到使用者 AppData 的 `com.vocalsync.studio/bin`

若你的公司網路擋了對外連線，建議先用系統套件管理器或可信來源安裝 yt-dlp / FFmpeg，再回到 Download tab 按「偵測本機工具」並信任。

#### 下載選項

| 欄位 | 說明 |
|---|---|
| **網址** | 貼上 YouTube 影片、播放清單或頻道連結。程式會自動偵測類型並顯示標籤 |
| **格式** | `mp3` / `m4a` / `wav`（純音訊）/ `video`（影片 + 音訊）/ `subtitle_only`（只抓字幕） |
| **畫質** | 僅在選 `video` 時顯示：`best` / 1080p / 720p / 480p / 360p |
| **字幕** | `none` / 繁中 / 簡中 / 英文 / 日文 / `all`。若 YouTube 沒提供該語言字幕，會嘗試自動翻譯版 |
| **輸出目錄** | 預設桌面。點「**瀏覽**」更改 |

下載中會顯示進度條、速度與剩餘時間（ETA）。隨時可按「**取消下載**」中止。

#### 下載完成後

若有抓到字幕檔（`.srt` / `.vtt` / `.lrc`），下方會出現字幕清單。點旁邊的「**載入為歌詞**」可直接送到錄音分頁的歌詞面板，省去再次選檔。

### 4.2 練唱伴奏匯入

**位置**：準備分頁 → 練唱伴奏區塊

點「**匯入伴奏**」選擇本機檔案，支援格式：

- **音訊**：WAV / MP3 / M4A / AAC / FLAC / OGG
- **影片**：MP4 / MKV / WebM（畫面會在錄音分頁左側顯示）

匯入成功後會顯示檔名、時長、類型（音訊 / 影片）。此時畫面下方會自動執行：

1. 嘗試偵測對應的目標旋律（同名 vocals 檔或 UVR 快取）
2. 若是影片檔，自動掃描內嵌字幕軌

### 4.3 歌詞與字幕

**位置**：準備分頁 → 歌詞 / 字幕區塊

支援的歌詞格式：

| 格式 | 來源 | 時間精度 |
|---|---|---|
| **LRC** | KRC 轉檔、手動製作 | 逐行 |
| **SRT** | YouTube 字幕、Netflix 字幕 | 逐行（開始/結束時間）|
| **VTT** | Web Video Text Tracks | 逐行 |
| **TXT** | 純文字歌詞 | 無時間軸（僅顯示）|

#### 三種載入方式

1. **手動載入**：點「**載入歌詞檔**」→ 選 LRC / SRT / VTT / TXT
2. **影片內嵌字幕**：匯入影片後自動偵測，出現「#0 zho · subrip」之類的按鈕，點一下即抓出來
3. **YouTube 下載字幕**：下載完成後點字幕清單的「載入為歌詞」

#### 即時同步顯示

錄音分頁右上方切換到「**歌詞**」面板，當前播放行會自動置中高亮。

若字幕檔的時間軸不準（例如 YouTube 自動字幕會滯後 1~2 秒），切換到「**歌詞同步編輯器**」可視覺化地批次平移。

### 4.4 目標旋律（音高參考線）

**位置**：準備分頁 → 目標旋律區塊

目標旋律在音高時間軸上以**灰藍色線條**顯示，是你練唱時對齊的參考。有兩種來源：

#### 來源 A：匯入乾淨人聲（**推薦**）

適用情況：你有原曲，想精準對齊原唱的音高。

**前置作業**：用 UVR5 / Moises / Demucs 等**外部工具**（程式本身不做原曲分離，避免法律問題）從原曲分離出 `vocals.wav`。

**操作**：點「**匯入乾淨人聲**」→ 選 `vocals.wav`。後端會：

1. 跑 PYIN 音高提取，產生每個時間點的音符
2. 自動與你的練唱伴奏做 cross-correlation 對齊，算出時間偏移

#### 來源 B：中央聲道消除（**快速但粗糙**）

適用情況：沒有分離工具、想快速看個大概。

點「**消人聲提取**」→ 程式對立體聲練唱伴奏做 L-R 差分（center-cancel），用剩下的訊號跑音高偵測。

**限制**：只對「人聲置中、伴奏置於左右聲道」的流行歌有效。單聲道檔或現場錄音的效果差。

#### 對齊微調

匯入 melody 後，若程式自動對齊不夠精準，下方會出現**精細微調滑桿**（±500 ms）。拖動時音高時間軸上的灰藍線會即時位移，找到對齊的感覺後鬆手即可。

**對齊信心度標籤**：

- 🟢 **高信心**：peak-to-mean ratio 很高，通常直接用就好
- 🟡 **中信心**：可能需要微調 ±50 ms
- 🔴 **低信心**：自動對齊可能失敗，建議手動調整或換來源

### 4.5 音訊裝置設定

**位置**：準備分頁 → 裝置選擇區塊

#### 輸入裝置

你的麥克風。常見選項：

- 內建麥克風：筆電鍵盤上方的麥克風陣列
- USB 麥克風：Blue Yeti、HyperX QuadCast、舒爾 MV7 等
- 音訊介面：PreSonus Quantum ES2、Focusrite Scarlett、MOTU M2 等

**建議**：盡量選擇「**獨占模式**」可用的裝置（例如 ASIO 介面），延遲較低且不易被其他程式搶走。

#### 輸出裝置

**務必選擇耳機**，不要用電腦內建喇叭。原因：

- 喇叭發出的伴奏會被麥克風收錄到錄音裡，形成雙重疊音
- 延遲更大、音高偵測更不準

#### 音高引擎

| 選項 | 說明 | 建議 |
|---|---|---|
| **auto** | 程式自動選：預設 CREPE，若載入失敗 fallback 到 PYIN | **推薦** |
| **crepe** | 強制用 CREPE 神經網路模型，精度高但需要 DirectML | 有獨顯時 |
| **yin** | 強制用 PYIN 傳統演算法，CPU 吃重低但精度略差 | 低階機器 |

CREPE 模型為 `models/crepe-tiny.onnx`（約 22 MB），程式啟動時載入到記憶體。

### 4.6 延遲校準

**位置**：準備分頁 → 延遲校準區塊

**為什麼需要校準**：

你的麥克風收音 → 音訊驅動 → 作業系統 → 本程式 → 音訊驅動 → 耳機發音，整條鏈路會有數十到數百毫秒的延遲。若不校準，錄出來的演唱會比伴奏**慢 50~200 ms**，音高曲線也會對不上歌詞。

#### 自動校準流程（拍手對拍法）

1. 戴好耳機，確保輸入/輸出裝置選對
2. 點「**開始校準**」按鈕，程式進入 1.5 秒準備期
3. **節拍器開始發出木魚聲**（BPM 70，共 8 拍）
4. **跟著節拍器拍手**（或拍桌面、對麥克風發「嗒」聲都行）
5. 前 2 拍是暖身、讓你抓到節奏感；後 6 拍才用來量測
6. 結束後顯示結果：「**最適延遲：123 ms（標準差 ±4.2 ms）**」

原理：程式把你拍手的時間點與節拍器發聲的時間點相比，中位數即是你的裝置延遲。至少要偵測到 **3 個有效拍**才算成功，若失敗會跳錯誤訊息提示你確認麥克風音量。

校準結果會自動儲存到設定，下次啟動不用重做。

#### 拍準的幾個訣竅

- **環境要安靜**：冷氣、電風扇的底噪會讓 onset detection 誤觸發
- **拍手要清脆**：手掌合擊比指尖敲桌聲音更乾脆、onset 時間點更明確
- **不要預判**：一聽到木魚聲才拍，不要想說「下一拍大概快到了先拍」，提前會被判離群值剔除
- **麥克風音量別太小**：拍手聲音要明顯蓋過底噪，程式才抓得到

#### 手動調整

若自動校準結果不理想（例如「只偵測到 2 拍有效回應」的錯誤、或標準差太大代表裝置不穩定），可用滑桿手動在 0~500 ms 間調整。

建議作法：錄一小段後回放比對，若感覺演唱**慢於**伴奏 → 增加延遲；**快於**伴奏 → 減少延遲。

### 4.7 試聽 · 錄音 · 回放

**位置**：錄音分頁 → 播放列（上方）

四個圓鈕負責完整的練唱流程：

| 按鈕 | 圖示 | 功能 |
|---|---|---|
| **試聽** | ▶（藍色圓鈕）| 純播放伴奏，不錄音。用來熟悉歌曲 |
| **暫停** | ‖（藍色圓鈕）| 暫停當前模式（試聽/錄音/回放），保留位置 |
| **停止** | ■（深色圓鈕）| 停止並回到 0 秒 |
| **錄音** | ●（紅色圓鈕）| 同步播放伴奏並錄麥克風 |

#### 狀態機邏輯

- **閒置 → 試聽**：按 ▶
- **閒置 → 錄音**：按 ●（會先清掉上次錄音的緩衝）
- **錄音中 → 暫停**：按 ‖
- **暫停中 → 繼續**：按 ▶（繼續剛才的模式，不會混用）
- **任何狀態 → 停止並歸零**：按 ■ 或 `Esc`

#### 續錄功能

錄音中暫停，拖進度條到不同位置再按 ●，會從新位置接續錄。若進度條**往前拖**（回到已錄過的時間），程式會跳出 dialog 確認，避免誤觸毀掉已錄內容。

#### 清除錄音

若對這次錄音不滿意想整段重錄，按控制列的「**清除錄音**」按鈕。會要求確認，避免誤觸。

#### 回放

錄完後按控制列的「**回放**」按鈕，程式會同時播放伴奏與你剛錄的人聲（依校準延遲自動補償），讓你立刻聽到對位效果。

### 4.8 音高時間軸

錄音分頁下半部 / 音高分頁全螢幕顯示。左右滾動跟隨播放位置，即時更新。

#### 線條顏色

| 顏色 | 代表 |
|---|---|
| **灰藍線** | 目標旋律（來自 vocals / center-cancel） |
| **金黃線** | 你的即時演唱音高 |
| **米色網格** | 半音刻度 + 八度標示（C4、C5 等）|
| **紅色垂直線** | 當前播放時間游標 |

#### 音高偵測機制

後端以 **16 ms** 為一幀跑 CREPE 或 PYIN，輸出頻率（Hz）→ 音符（含 cents 偏移）。只在 RMS 大於門檻、且模型信心度（CREPE）或清晰度（PYIN）夠高時才畫線，避免氣音與底噪造成亂跳。

#### 延遲補償

畫面上看到的即時音高會**自動補償校準延遲**，所以你的線條與歌詞、目標旋律的對齊點就是**實際同步點**，可以直接判斷「唱準了沒」。

### 4.9 A-B 循環

**位置**：錄音分頁 → 控制列 A-B 按鈕 或 快捷鍵 `A` / `B`

針對副歌、難句反覆練習。操作流程：

1. 播放到想循環的**起點** → 按 `A` 或 A 按鈕（亮藍色）
2. 播放到想循環的**終點** → 按 `B` 或 B 按鈕（亮藍色，且 A 已設才可按）
3. 進度條上會出現藍色區域，自動在此區段內循環
4. 想取消循環 → 按 `✕` 按鈕

A-B 循環可以與錄音、試聽、回放任一模式並用。

### 4.10 變速 · 變調

**位置**：錄音分頁 → 控制列 Speed / Pitch 按鈕

#### 變速（不改變音高）

後端走 **WSOLA**（Waveform Similarity Overlap-Add）時間拉伸演算法。六段檔位：

`0.5x / 0.75x / 0.9x / 1.0x / 1.1x / 1.25x`

- **慢速練習高難度樂句**：調到 0.75x 或 0.5x，學旋律起伏
- **快速瀏覽長曲**：調到 1.25x

1.0x 走 bypass（零延遲直通），其他檔位經 WSOLA 處理會有輕微音質損失。

#### 變調（移調）

不改變速度、只升降音高。範圍 -12 ~ +12 半音（上下 1 個八度）。

**用法**：

- 原曲太高唱不上去 → 按 `-` 往下移幾個半音（一般 2~3 個）
- 想練轉調版本 → 按 `+` 往上移

移調與變速可以同時使用，但**越偏離 1.0x + 0 半音，音質損失越明顯**。

#### 重設按鈕（↺）

每個控制右邊都有重設按鈕，一鍵回到預設值（1.0x / 0 半音）。

### 4.11 匯出錄音

**位置**：錄音分頁 → 控制列「**匯出**」按鈕

只要錄過至少一次（即使錄完又清除過），「匯出」按鈕就會亮起。

#### 匯出內容

點下後選擇目的資料夾與檔名前綴，程式會輸出**兩個 WAV 檔**：

| 檔名 | 內容 | 用途 |
|---|---|---|
| `<前綴>_vocal.wav` | 純人聲軌（無伴奏）| 上傳混音工作室、再製作 |
| `<前綴>_mix.wav` | 伴奏 + 人聲 混音 | 直接發佈、分享給朋友聽 |

#### 自動音量平衡（選用）

控制列右下有「**自動音量平衡**」checkbox。勾選後匯出的 `_mix.wav` 會以 LUFS-ish 規則自動平衡伴奏與人聲比例，避免「伴奏蓋過人聲」或反之。

若想要完全手動控制混音，取消勾選即可，匯出時會直接用你在控制列調好的音量比例。

## 6. 常見問題 FAQ

### Q1. 錄出來的聲音有回音 / 雙重音？

**A**：幾乎 100% 是因為你沒戴耳機或耳機漏音，麥克風把喇叭/耳機流出的伴奏也收進去了。改用**封閉式耳機**。

### Q2. 音高曲線一直亂跳，不像我唱的？

**A**：可能原因：

1. **麥克風增益太高**：控制列 Mic 滑桿降到 100% 以下，觀察音量條紅色區的比例，盡量讓巔峰落在黃綠色區
2. **底噪太大**：房間開冷氣、電風扇會讓 RMS 門檻誤觸發。關掉或換安靜環境
3. **音高引擎用錯**：低階機器選到 CREPE 跑不動、卡幀。改選 `yin`

### Q3. 下載 YouTube 時顯示「HTTP Error 403 Forbidden」或「Sign in to confirm you're not a bot」？

**A**：YouTube 有時會擋 yt-dlp 的請求。解法：

1. 在下載區塊的工具狀態列看 yt-dlp 版本，若過舊點「**自動安裝**」更新到最新版
2. 若還是不行，可能要等 yt-dlp 社群修 bug（通常一週內）
3. 或改用外部 yt-dlp 下載後再用「匯入伴奏」載入

### Q4. 我的電腦沒有獨顯，CREPE 會卡嗎？

**A**：CREPE-tiny 模型已優化為**低階 CPU 也能跑**（約吃 15~25% CPU）。若你發現卡頓或跟不上，改成 `yin` 引擎即可，音高精度差一些但 CPU 負擔極低。

### Q5. 匯出的 WAV 檔可以給其他程式處理嗎？

**A**：可以。匯出格式是 **16-bit PCM WAV、取樣率跟伴奏一致**（通常 44.1 kHz 或 48 kHz），任何 DAW（Reaper、FL Studio、Logic、Audition）都讀得到。

### Q6. 支援 macOS 或 Linux 嗎？

**A**：原始碼本身多平台（Tauri v2 + Rust），理論上可在 macOS / Linux 建置。但官方**尚未提供 portable release**、也未做實機測試。有興趣的話可以 fork 後自行 `npm run tauri build`，歡迎把結果回報到 Issues。

### Q7. 這個程式會上傳我的錄音嗎？

**A**：**不會**。程式完全離線運作，除了從 YouTube 下載伴奏時會發網路請求外，錄音、音高分析、匯出全都在本機完成。你匯出的 WAV 檔也只存到你指定的目錄，不會上傳任何地方。

### Q8. 可以商業用嗎？

**A**：VocalSync Studio 本身是 MIT License，程式本體可自由使用（包括商業）。但：

- 透過 yt-dlp 下載的內容要遵守**原始內容的著作權**與 YouTube 服務條款
- 你錄唱的歌曲若要公開發佈（上傳到 YouTube、Spotify），需取得原曲的**翻唱授權**（KTV 系統的授權範圍不含公開發行）

### Q9. 可以回報 bug 或提需求嗎？

**A**：超歡迎！見下方「[回報問題 · 支持專案](#8-回報問題--支持專案)」。

## 7. 故障排除

### 程式無法啟動

| 症狀 | 可能原因 | 解法 |
|---|---|---|
| 雙擊沒反應 | WebView2 runtime 缺失（Windows 10 早期版本）| 從 [Microsoft 官網](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) 下載並安裝 |
| 「找不到 DirectML.dll」 | 資料夾結構被改動 | 重新解壓整個 zip，不要單獨搬 exe |
| SmartScreen 警告 | 程式沒 code-signing | 點「其他資訊」→「仍要執行」 |

### 錄音沒有聲音

1. 檢查「準備」分頁 → 裝置選擇 → 輸入裝置是否選到正確的麥克風
2. 控制列 Mic 滑桿是否拉到 0
3. Windows 設定 → 隱私權 → 麥克風 → 確認應用程式可存取麥克風
4. 其他軟體（Discord、OBS）是否獨占了麥克風

### 音高曲線不顯示

- 麥克風音量太小，連 RMS 門檻都觸不到 → 提高 Mic gain
- 音高引擎載入失敗（CREPE 找不到 `models/crepe-tiny.onnx`）→ 重新解壓 portable 包

### 對齊一直失敗

- 練唱伴奏與 vocals 來源差太多（不同版本、remix 等）→ 手動用微調滑桿調到耳朵聽起來對齊為止
- 來源音量過小 → 確認 vocals.wav 不是靜音的空檔

### 變速後聲音破破的

- 檔位越極端（0.5x 或 1.25x）音質越差，這是 WSOLA 的固有限制
- 若要高品質慢速練習，建議用 Reaper / Audition 事先做 time-stretch 再匯入
