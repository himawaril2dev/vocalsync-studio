## 1. はじめに

VocalSync Studio は**デスクトップ版の練習ルーム**です。歌の練習をしたい方に次のような機能を提供します：

- 伴奏を聴きながら録音し、自分の歌声のピッチカーブをその場で確認
- 目標メロディ（原曲から分離したボーカル）と並べて比較
- A-B ループで特定の部分を繰り返し練習
- ピッチを変えずにテンポだけ落として難しいフレーズを練習

アプリは**完全にオフライン動作**（伴奏ダウンロード時を除く）で、録音データは自分の PC にのみ保存されます。クラウドへのアップロードは一切ありません。

> 📢 **透明性の表明**
> 作者はプログラミングが本業ではなく、本プロジェクトは AI（Claude / Codex）との協同で作られています。すべての機能は実機で検証し、複数モデルによるコードレビューを経ていますが、導入前にご自身のユースケースでリスクを判断してください。

## 2. 5分クイックスタート

### Step 1｜アプリを起動

`VocalSync.Studio.Portable.x.y.z.zip` を解凍し、フォルダ内の **`vocalsync-studio.exe`** をダブルクリックします。

> ⚠️ フォルダ構成を変更しないでください。`DirectML.dll`、`models/` は同梱された実行時依存で、**単独で移動できません**。自動インストールされた yt-dlp / FFmpeg はポータブル版のルートを優先するため、フォルダごとならどこに置いても動作します。

### Step 2｜伴奏を用意

**オプション A：YouTube からダウンロード**

1. 左側のナビで「**準備**」タブを選択
2. 一番上の「**YouTube ダウンロード**」セクションを展開
3. YouTube の URL を貼り付け（例：`https://www.youtube.com/watch?v=xxxxxxxxxxx`）
4. 形式を `mp3` に選び、「**ダウンロード**」をクリック

初回に yt-dlp や FFmpeg が見つからない旨のメッセージが出たら、「**自動インストール**」ボタンをクリックすればアプリが用意してくれます。

**オプション B：ローカルの音声ファイルを使う**

「**練習用伴奏**」セクションを展開 → 「**伴奏を読み込む**」をクリック → MP3 / WAV / M4A などのファイルを選択。

### Step 3｜入出力デバイスを設定

「**デバイス選択**」セクションを展開：

- **入力デバイス**：マイク
- **出力デバイス**：ヘッドホン（**録音時はヘッドホンを強く推奨**します。そうしないとマイクが伴奏を拾い、録音がこもってしまいます）

### Step 4｜レイテンシのキャリブレーション（初回推奨）

「**レイテンシキャリブレーション**」セクションを展開 → 「**キャリブレーション開始**」をクリック → ヘッドホンを装着し、**メトロノームの木魚音に合わせて手拍子**してください（机を叩いたり、マイクに向かって「タ」と言うのでも OK）。

アプリは 8 拍（前半 2 拍はウォームアップ + 後半 6 拍が測定）を鳴らし、手拍子のタイミングとメトロノームのズレを検出して最適な補償値を自動算出・保存します。デバイスを変えない限り再実行は不要です。

### Step 5｜録音開始

1. 「**録音**」タブに切り替え
2. スペースキーまたは**赤い録音ボタン**を押して開始
3. 伴奏に合わせて歌う
4. スペースで一時停止、`Esc` で停止
5. 「**エクスポート**」で WAV ファイルとして保存

画面下部ではピッチカーブがリアルタイムで表示され、上部では歌詞や映像が同期して再生されます。

## 3. メイン画面ガイド

左側のナビには 4 つのタブがあります。

### 🎯 準備（Setup）

初回起動時に開かれるタブ。**「録音前に準備しておくこと」**はすべてここに折りたたみセクションとしてまとまっています：

- YouTube ダウンロード
- 練習用伴奏
- 歌詞 / 字幕
- 目標メロディ
- デバイス選択
- レイテンシキャリブレーション

セクションの開閉状態はタブを切り替えても保持されるので、毎回開き直す必要はありません。

### 🎤 録音（Recording）

メインの作業エリア。上半分は左右 2 カラム（区切りをドラッグで比率調整可）：

- **左**：映像 / 音声ファイル情報（映像ファイルを読み込んだ場合は映像を表示）
- **右**：歌詞パネル / 歌詞同期エディタ（切り替え可能）

下半分は**全幅のピッチタイムライン**で、歌唱と目標メロディの対比をリアルタイム表示します。

最下部にはコントロール行が 2 段：

- **トランスポート行**：試聴 / 一時停止 / 停止 / 録音 / プログレスバー / A-B ループ
- **コントロール行**：伴奏音量 / マイク音量 / 速度 / ピッチ / エクスポート / 再生

### 📊 ピッチ（Pitch）

ピッチ解析に特化したタブ。上部に現在のメロディ状態と整列結果、中央は全画面のピッチタイムライン、下部にはミニトランスポートがあり、再生しながら観察できます。

録音せず、前回の録音カーブと目標メロディの対比だけを見直したいときに便利です。

### ℹ️ アプリ情報（About）

バージョン情報、透明性の表明、ライセンス、支援リンク。

### 言語切替

左側ナビの最下部に言語スイッチャーがあり、**繁體中文 / English / 日本語** を即時切り替えできます。再起動は不要です。

## 4. 機能詳細

### 4.1 YouTube 伴奏ダウンロード

**場所**：準備タブ → YouTube ダウンロードセクション

yt-dlp と FFmpeg は**同梱されていません**。初回利用時にステータスが表示されます：

| ステータス | 意味 | 対応 |
|---|---|---|
| 🟢 yt-dlp v2026.x.x | インストール済み | 不要 |
| 🔴 yt-dlp 未インストール | yt-dlp が見つからない | 「**自動インストール**」ボタンをクリック |
| 🟢 FFmpeg ✓ | システム側で使用可能 | 不要 |
| 🔴 FFmpeg 未インストール | システム側にない | 「**自動インストール**」ボタンをクリック |

**ローカル信頼と自動インストールの動作**：

- **ローカルツールを使う**：PC に入っている yt-dlp / FFmpeg を検出できます。信頼するとパスと SHA-256 を記録し、以後は同じ検証済みファイルだけを実行します。
- **yt-dlp 自動インストール**：GitHub Releases から `yt-dlp.exe` をダウンロードし、SHA-256 検証後にポータブル版のルートへ優先配置します。
- **FFmpeg 自動インストール**：固定された essentials build をダウンロードし、SHA-256 検証後に `ffmpeg.exe` / `ffprobe.exe` を展開してポータブル版のルートへ優先配置します。
- **フォールバック先**：ポータブル版のルートに書き込めない場合は、ユーザー AppData の `com.vocalsync.studio/bin` に配置します。

社内ネットワークが外部接続をブロックしている場合は、信頼できる方法で yt-dlp / FFmpeg を先にインストールし、Download tab でローカルツールを検出して信頼してください。

#### ダウンロードオプション

| 項目 | 説明 |
|---|---|
| **URL** | YouTube 動画 / プレイリスト / チャンネルのリンクを貼り付け。アプリがタイプを自動判定してタグを表示 |
| **形式** | `mp3` / `m4a` / `wav`（音声のみ）/ `video`（映像 + 音声）/ `subtitle_only`（字幕のみ） |
| **画質** | `video` を選んだときだけ表示：`best` / 1080p / 720p / 480p / 360p |
| **字幕** | `none` / 繁体字 / 簡体字 / 英語 / 日本語 / `all`。該当言語字幕がない場合は自動翻訳版にフォールバック |
| **出力フォルダ** | デフォルトはデスクトップ。「**参照**」で変更可 |

ダウンロード中はプログレスバー、速度、残り時間（ETA）が表示されます。「**ダウンロードをキャンセル**」でいつでも中断できます。

#### ダウンロード完了後

字幕ファイル（`.srt` / `.vtt` / `.lrc`）を取得できた場合は、下部に字幕リストが表示されます。各エントリ横の「**歌詞として読み込む**」をクリックすると、録音タブの歌詞パネルへ直接送られます（再選択不要）。

### 4.2 練習用伴奏の読み込み

**場所**：準備タブ → 練習用伴奏セクション

「**伴奏を読み込む**」でローカルファイルを選択します。対応形式：

- **音声**：WAV / MP3 / M4A / AAC / FLAC / OGG
- **映像**：MP4 / MKV / WebM（映像は録音タブの左側に表示されます）

読み込み成功後にファイル名、再生時間、種別（音声 / 映像）が表示されます。同時に次の処理が自動実行されます：

1. 対応する目標メロディを自動検出（同名の vocals ファイルまたは UVR キャッシュ）
2. 映像ファイルなら内蔵字幕トラックをスキャン

### 4.3 歌詞と字幕

**場所**：準備タブ → 歌詞 / 字幕セクション

対応している歌詞フォーマット：

| 形式 | 入手元 | 時間精度 |
|---|---|---|
| **LRC** | KRC 変換、手動作成 | 行単位 |
| **SRT** | YouTube 字幕、Netflix 字幕 | 行単位（開始/終了） |
| **VTT** | Web Video Text Tracks | 行単位 |
| **TXT** | プレーンテキスト歌詞 | 時間軸なし（表示のみ） |

#### 読み込み方法（3 種類）

1. **手動読み込み**：「**歌詞ファイルを読み込む**」をクリック → LRC / SRT / VTT / TXT を選択
2. **映像の内蔵字幕**：映像を読み込むと自動検出され、「#0 zho · subrip」のようなボタンが現れるのでクリック
3. **YouTube でダウンロードした字幕**：ダウンロード後、字幕リストの「歌詞として読み込む」をクリック

#### リアルタイム同期表示

録音タブ右上を「**歌詞**」パネルに切り替えると、現在再生中の行が自動で中央に寄り、ハイライトされます。

字幕ファイルの時間軸がずれている（YouTube 自動字幕は 1〜2 秒遅れやすい）場合は、「**歌詞同期エディタ**」に切り替えて視覚的に一括シフトできます。

### 4.4 目標メロディ（ピッチ参考ライン）

**場所**：準備タブ → 目標メロディセクション

目標メロディはピッチタイムライン上に**グレーブルーのライン**として表示され、練習時の整合基準になります。ソースは 2 種類：

#### ソース A：クリーンなボーカルを読み込む（**推奨**）

向いているケース：原曲があり、オリジナルボーカルのピッチに正確に合わせたい。

**事前準備**：UVR5 / Moises / Demucs などの**外部ツール**で `vocals.wav` を分離してください（法的リスクを避けるため、アプリ自体は音源分離を行いません）。

**手順**：「**クリーンなボーカルを読み込む**」→ `vocals.wav` を選択。バックエンドは以下を実行します：

1. PYIN ピッチ抽出で各時点の音符を生成
2. 練習用伴奏とクロスコリレーションで自動整列し、時間オフセットを算出

#### ソース B：センターチャンネル除去（**簡便だが精度は粗い**）

向いているケース：分離ツールがなく、ざっくり参考ラインが欲しい。

「**ボーカル除去で抽出**」をクリック → ステレオ伴奏に L-R 差分（center-cancel）をかけ、残った信号でピッチ検出を実行します。

**制限**：「ボーカルがセンター、伴奏が左右」という典型的なポップスにしか効きません。モノラル音源やライブ録音では精度が大きく落ちます。

#### 整列の微調整

メロディ読み込み後、自動整列が十分でない場合は**微調整スライダー**（±500 ms）が表示されます。ドラッグすると、グレーブルーのラインがリアルタイムで移動するので、耳で合うポイントを探して指を離してください。

**整列信頼度ラベル**：

- 🟢 **高信頼**：peak-to-mean ratio が高く、通常そのまま使えます
- 🟡 **中信頼**：±50 ms 程度の微調整が必要かもしれません
- 🔴 **低信頼**：自動整列が失敗している可能性。手動調整またはソース変更を推奨

### 4.5 オーディオデバイス設定

**場所**：準備タブ → デバイス選択セクション

#### 入力デバイス

マイクです。代表的な選択肢：

- 内蔵マイク：ノート PC キーボード上部のマイクアレイ
- USB マイク：Blue Yeti、HyperX QuadCast、Shure MV7 など
- オーディオインターフェース：PreSonus Quantum ES2、Focusrite Scarlett、MOTU M2 など

**おすすめ**：できれば「**排他モード**」対応デバイス（例：ASIO インターフェース）を選んでください。レイテンシが低く、他のアプリに奪われにくいです。

#### 出力デバイス

**必ずヘッドホンを選んでください**。PC 内蔵スピーカーは NG です。理由：

- スピーカーから出る伴奏をマイクが拾い、二重音になります
- レイテンシが増え、ピッチ検出の精度も落ちます

#### ピッチエンジン

| 選択肢 | 説明 | 推奨 |
|---|---|---|
| **auto** | アプリ自動選択：デフォルトは CREPE、ロードに失敗したら PYIN にフォールバック | **推奨** |
| **crepe** | CREPE ニューラルネットモデルを強制使用。高精度だが DirectML が必要 | 専用 GPU があるとき |
| **yin** | 従来の PYIN アルゴリズムを強制。CPU 負荷が低く、精度はやや劣る | 低スペック機 |

CREPE モデルは `models/crepe-tiny.onnx`（約 22 MB）で、起動時にメモリへ読み込まれます。

### 4.6 レイテンシキャリブレーション

**場所**：準備タブ → レイテンシキャリブレーションセクション

**なぜキャリブレーションが必要か**：

マイク入力 → オーディオドライバ → OS → 本アプリ → オーディオドライバ → ヘッドホン、という経路には数十〜数百ミリ秒の遅延が累積します。キャリブレーションしないと、録音された歌声が伴奏より**50〜200 ms 遅れ**、ピッチカーブも歌詞とずれます。

#### 自動キャリブレーションの流れ（クラップ同期法）

1. ヘッドホンを装着し、入出力デバイスが正しく選ばれていることを確認
2. 「**キャリブレーション開始**」をクリック → 1.5 秒の準備期間に入る
3. **メトロノームが木魚音を鳴らし始めます**（BPM 70、全 8 拍）
4. **メトロノームに合わせて手拍子**（机を叩く、マイクに「タ」と発声でも OK）
5. 前 2 拍はウォームアップでテンポに慣れる時間。後 6 拍が測定対象
6. 終了後に結果を表示：「**最適レイテンシ：123 ms（標準偏差 ±4.2 ms）**」

仕組み：手拍子のタイミングとメトロノームの発音タイミングを比較し、その中央値をデバイスのレイテンシとします。成功には少なくとも **3 拍の有効検出**が必要で、失敗した場合はマイク音量の確認を促すエラーが表示されます。

キャリブレーション結果は設定に自動保存され、次回起動時に再実行する必要はありません。

#### 手拍子のコツ

- **静かな環境で**：エアコンやファンのノイズが onset detection を誤爆させる
- **クリスプに叩く**：手のひら同士の拍手の方が指先で机を叩くよりオンセットが明確
- **先走らない**：木魚音を聞いてから叩く。予測して早めに叩くと外れ値として除外される
- **マイク音量を小さくしすぎない**：手拍子音がノイズフロアより十分大きい必要がある

#### 手動調整

自動結果が理想的でない場合（例：「有効な拍が 2 つしか検出できません」、標準偏差が大きすぎるなど）、スライダーで 0〜500 ms の範囲で手動調整できます。

推奨ワークフロー：短い録音をしてから再生比較し、歌声が伴奏より**遅れて**聞こえる → レイテンシを増やす、**先行して**聞こえる → 減らす。

### 4.7 試聴 · 録音 · 再生

**場所**：録音タブ → トランスポート行（上段）

4 つの丸ボタンで練習フローをカバーします：

| ボタン | アイコン | 機能 |
|---|---|---|
| **試聴** | ▶（青） | 伴奏のみ再生、録音なし。曲に慣れるため |
| **一時停止** | ‖（青） | 現在のモード（試聴/録音/再生）を一時停止し、位置を保持 |
| **停止** | ■（濃色） | 停止して 0 秒に戻る |
| **録音** | ●（赤） | 伴奏を再生しながらマイクを録音 |

#### ステートマシン

- **アイドル → 試聴**：▶ を押す
- **アイドル → 録音**：● を押す（前回の録音バッファは自動クリア）
- **録音中 → 一時停止**：‖ を押す
- **一時停止中 → 再開**：▶ を押す（直前のモードを継続。モード混在なし）
- **任意の状態 → 停止して先頭へ**：■ または `Esc`

#### 追加録音機能

録音中に一時停止し、プログレスバーをドラッグして別位置で ● を押すと、その位置から追加録音を継続します。ただし**過去の位置に戻す**（既に録音された時刻へ）場合は、誤操作による上書き防止のため確認ダイアログが表示されます。

#### 録音クリア

今回の録音に満足できず全部やり直したいときは、コントロール行の「**録音クリア**」ボタンを押してください。誤操作防止のため確認が入ります。

#### 再生

録音後、コントロール行の「**再生**」を押すと、伴奏と録音した歌声が同時に（キャリブレーションされたレイテンシで自動補正されつつ）流れるので、整合度をすぐ確認できます。

### 4.8 ピッチタイムライン

録音タブの下半分 / ピッチタブの全画面で表示されます。再生位置に合わせて左右にスクロールし、リアルタイム更新されます。

#### ライン色

| 色 | 意味 |
|---|---|
| **グレーブルー** | 目標メロディ（vocals / center-cancel 由来） |
| **ゴールド** | リアルタイムの自分のピッチ |
| **ベージュグリッド** | 半音スケール + オクターブ表記（C4、C5 など） |
| **赤い縦線** | 現在の再生カーソル |

#### ピッチ検出の仕組み

バックエンドは **16 ms** フレームで CREPE または PYIN を実行し、周波数（Hz）→ 音符（cents オフセット付き）に変換します。RMS が閾値を超え、かつモデル信頼度（CREPE）または明瞭度（PYIN）が十分高いフレームだけを描画するため、息や環境音でラインが暴れません。

#### レイテンシ補償

画面に表示されるリアルタイムピッチは**キャリブレーションされたレイテンシが既に補正されている**ため、ライン・歌詞・目標メロディの整列位置はそのまま**実際の同期ポイント**です。「音が合っているか」をその場で判断できます。

### 4.9 A-B ループ

**場所**：録音タブ → コントロール行の A-B ボタン、またはショートカット `A` / `B`

サビや難しい部分を繰り返し練習するための機能。操作フロー：

1. ループの**開始点**まで再生 → `A` または A ボタンを押す（青く光る）
2. ループの**終了点**まで再生 → `B` または B ボタンを押す（青く光る。A 設定後のみ有効）
3. プログレスバーに青い区間が表示され、その範囲で自動ループ
4. キャンセルしたいとき → `✕` ボタンをクリック

A-B ループは録音・試聴・再生のどのモードとも併用可能です。

### 4.10 速度 · ピッチシフト

**場所**：録音タブ → コントロール行の Speed / Pitch ボタン

#### 速度（ピッチ固定）

バックエンドは **WSOLA**（Waveform Similarity Overlap-Add）タイムストレッチアルゴリズムを使用。6 段階の固定値：

`0.5x / 0.75x / 0.9x / 1.0x / 1.1x / 1.25x`

- **難しいフレーズをゆっくり練習**：0.75x や 0.5x で旋律の起伏を把握
- **長い曲を素早くブラウズ**：1.25x

1.0x は bypass（ゼロレイテンシ直通）。他の速度では WSOLA 処理によるわずかな音質劣化があります。

#### ピッチシフト（移調）

速度は変えず、音高だけを上下させます。範囲は -12〜+12 半音（上下 1 オクターブ）。

**使い方**：

- 原曲が高すぎて歌えない → `-` で数半音下げる（通常 2〜3 半音）
- 転調バージョンを練習したい → `+` で上げる

ピッチシフトと速度変更は併用できますが、**1.0x + 0 半音から離れるほど音質劣化が目立ちます**。

#### リセットボタン（↺）

各コントロールの右にリセットボタンがあり、ワンクリックでデフォルト（1.0x / 0 半音）に戻せます。

### 4.11 録音のエクスポート

**場所**：録音タブ → コントロール行の「**エクスポート**」ボタン

一度でも録音していれば（その後クリアしていてもバッファ上の最終テイクが残っている限り）「エクスポート」ボタンが有効になります。

#### エクスポートされる内容

出力フォルダとファイル名プレフィックスを指定すると、アプリは **2 つの WAV ファイル**を書き出します：

| ファイル名 | 内容 | 用途 |
|---|---|---|
| `<プレフィックス>_vocal.wav` | 純粋な歌声トラック（伴奏なし） | ミキシングスタジオへの納品、再加工 |
| `<プレフィックス>_mix.wav` | 伴奏 + 歌声のミックス | そのまま公開、友人にシェア |

#### 自動音量バランス（任意）

コントロール行右下に「**正規化**」チェックボックスがあります。オンにすると、エクスポートされる `_mix.wav` は LUFS 的なルールで伴奏と歌声のバランスを自動調整し、「伴奏が歌声を覆い隠す／その逆」を防ぎます。

完全に手動でミックスを制御したい場合はチェックを外してください。エクスポート時には、コントロール行で設定した音量比率がそのまま反映されます。

| キー | 機能 | タイミング |

## 6. よくある質問 FAQ

### Q1. 録音にエコーや二重音が入ります

**A**：ほぼ確実にヘッドホンを着けていないか、音漏れで伴奏がマイクに入り込んでいます。**密閉型ヘッドホン**に切り替えてください。

### Q2. ピッチカーブが激しく暴れて、自分の歌と合いません

**A**：考えられる原因：

1. **マイクゲインが高すぎる**：コントロール行の Mic スライダーを 100% 未満に下げ、メーターを見ながらピークが黄緑ゾーンに収まるよう調整
2. **ノイズフロアが高い**：エアコンやファンのノイズが RMS 閾値を越えてしまう。電源を切るか、静かな環境へ
3. **ピッチエンジンの選択ミス**：低スペック機で CREPE が追いつかずコマ落ちする場合がある。`yin` に切り替え

### Q3. YouTube ダウンロードで「HTTP Error 403 Forbidden」や「Sign in to confirm you're not a bot」が出ます

**A**：YouTube が時々 yt-dlp のリクエストをブロックします。対応方法：

1. ダウンロードセクションのツールステータスで yt-dlp バージョンを確認。古ければ「**自動インストール**」で最新版へ更新
2. それでもダメな場合は yt-dlp コミュニティの修正待ち（通常 1 週間以内）
3. または外部で yt-dlp を使って先にダウンロードし、「**伴奏を読み込む**」で読み込む

### Q4. 専用 GPU がないのですが CREPE はカクつきますか？

**A**：CREPE-tiny モデルは**低スペック CPU でも動作するよう最適化**されています（CPU 使用率は 15〜25% 程度）。それでもカクつきやフレーム落ちを感じる場合は `yin` エンジンに切り替えてください。ピッチ精度はやや下がりますが、CPU 負荷は大幅に減ります。

### Q5. エクスポートされた WAV を他のソフトで処理できますか？

**A**：できます。エクスポート形式は **16-bit PCM WAV、伴奏と同じサンプリングレート**（通常 44.1 kHz または 48 kHz）で、どの DAW（Reaper、FL Studio、Logic、Audition）でも開けます。

### Q6. macOS や Linux に対応していますか？

**A**：ソースコード自体はクロスプラットフォーム（Tauri v2 + Rust）なので、理論的には macOS / Linux でもビルド可能です。ただし**公式の portable release は提供しておらず**、実機テストもしていません。興味がある方はリポジトリを fork して `npm run tauri build` を実行し、結果を Issues で共有していただけると嬉しいです。

### Q7. このアプリは録音データをアップロードしますか？

**A**：**しません**。アプリは完全オフラインです。YouTube から伴奏をダウンロードするときだけネットワーク通信が発生し、録音・ピッチ解析・エクスポートはすべてローカルで完結します。エクスポートされた WAV ファイルは指定したフォルダにのみ保存され、どこにもアップロードされません。

### Q8. 商用利用できますか？

**A**：VocalSync Studio 本体は MIT License なので、アプリ自体は自由に使えます（商用を含む）。ただし：

- yt-dlp でダウンロードしたコンテンツは**元コンテンツの著作権**と YouTube 利用規約に従う必要があります
- 録音した歌を公開する（YouTube、Spotify などへの投稿）場合、原曲の**カバー配信ライセンス**が必要です（カラオケ／KTV のライセンスは公開配信を含みません）

### Q9. バグ報告や機能要望はできますか？

**A**：大歓迎です！下の「[問題報告 · プロジェクト支援](#8-問題報告--プロジェクト支援)」をご覧ください。

## 7. トラブルシューティング

### アプリが起動しない

| 症状 | 考えられる原因 | 解決方法 |
|---|---|---|
| ダブルクリックしても反応なし | WebView2 ランタイム不足（Windows 10 初期ビルド） | [Microsoft 公式サイト](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) からダウンロード・インストール |
| 「DirectML.dll が見つからない」 | フォルダ構成が変更されている | zip を丸ごと再解凍する。exe だけを動かさない |
| SmartScreen 警告 | アプリがコード署名されていない | 「詳細情報」→「実行」をクリック |

### 録音に音が入らない

1. 準備タブ → デバイス選択 → 正しいマイクが選択されているか
2. コントロール行の Mic スライダーが 0 になっていないか
3. Windows 設定 → プライバシー → マイク → アプリのアクセスが許可されているか
4. 他のソフト（Discord、OBS）がマイクを排他的に占有していないか

### ピッチカーブが表示されない

- マイク音量が小さすぎて RMS 閾値に届かない → Mic ゲインを上げる
- ピッチエンジンの読み込み失敗（CREPE が `models/crepe-tiny.onnx` を見つけられない）→ portable zip を再解凍

### 整列が常に失敗する

- 練習用伴奏と vocals のソースが違いすぎる（別バージョン、リミックス等）→ 微調整スライダーで耳頼りに合わせる
- ソース音量が小さすぎる → `vocals.wav` が無音になっていないか確認

### 速度変更後に音がガサガサする

- 極端な速度（0.5x や 1.25x）ほど音質が下がるのは WSOLA の原理的な限界
- 高音質でゆっくり練習したいなら、Reaper / Audition などで事前にタイムストレッチして読み込むのがおすすめ
